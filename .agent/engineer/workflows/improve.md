# Improve

> 새 기능보다는 UX, 정보 구조, 접근성, 가독성을 개선할 때 쓴다.

---

## 절차

1. 현재 불편함이나 마찰 지점을 구체화한다.
2. 필요하면 `Designer ui-review` 결과를 먼저 반영한다.
3. 기존 시각 언어를 유지하면서 개선안을 구현한다.
4. locale 길이 차이와 모바일 폭을 우선 확인한다.
5. lint/build 검증 후 before/after 차이를 정리한다.

---

## 자주 다루는 개선 범위

- Resume 정보 밀도 조절
- About 페이지 소개 위계 정리
- Header 네비게이션 사용성
- 링크/버튼 접근성
- SEO 메타 보완
