# Review

`origin/main` 대비 변경사항을 리뷰한다.
기계적 이슈는 자동 수정하고, 판단이 필요한 항목만 보고한다.

## 인자

인자 없이 실행한다.

## 절차

### 1. diff 수집

```bash
git fetch origin
git diff origin/main --name-only
git diff origin/main --stat
```

### 2. Fix-First 자동 수정 (AUTO-FIX)

아래 기계적 이슈를 **승인 없이 자동 수정**한다:

| # | 항목 | 처리 |
|---|------|------|
| F1 | 파일 끝 개행 누락 | 자동 추가 |
| F2 | TypeScript 타입 에러 | `npx tsc --noEmit` 실행 후 수정 |
| F3 | 누락/불필요한 import | 자동 추가/제거 |
| F4 | ESLint 위반 | `npx eslint --fix` 실행 |

수정한 항목을 "AUTO-FIX 완료" 헤더로 보고한다.

### 3. ASK 분석 (판단 필요 항목)

아래 관점에서 분석하고, 발견된 항목을 번호 매겨 보고한다:

#### A. 코드 품질

| 관점 | 체크 항목 |
|------|----------|
| **DRY** | 동일 로직이 반복되는가? 사전 파일(ko/ja/en) 간 구조 불일치가 있는가? |
| **일관성** | 3개 언어 사전의 키/구조가 동기화되어 있는가? |
| **타입 안전** | Dictionary 타입 정의와 실제 데이터가 일치하는가? |

#### B. UI/UX

| 관점 | 체크 항목 |
|------|----------|
| **반응형** | 모바일/데스크톱에서 레이아웃이 깨지지 않는가? |
| **접근성** | 시맨틱 HTML, alt 텍스트, 색상 대비가 적절한가? |
| **SEO** | meta 태그, OG 태그, 구조화 데이터가 적절한가? |

#### C. 보안

- XSS 취약점 (dangerouslySetInnerHTML 사용 등)
- 민감 정보 하드코딩 (.env, API 키)

### 4. 보고

```
## AUTO-FIX 완료
- F1: 파일 끝 개행 추가 (N개 파일)
- F4: ESLint 자동 수정 (N개 파일)

## ASK (판단 필요)
1. [파일:라인] 현재 문제 → 개선 제안
2. ...

## 리뷰 요약
- 위험도: 낮음/중간/높음
- 변경 파일: N개
- AUTO-FIX: N개
- ASK: N개
```

- ASK 항목이 없으면 **"추가 개선 포인트 없음"**을 보고.
- 있으면 번호 선택을 요청한다 (예: "1, 2 진행해"). 승인 항목만 구현.
- 과도한 엔지니어링을 피한다.

## 규칙

- **읽기 + 기계적 자동 수정만** — ASK 항목은 사용자 승인 전까지 수정하지 않는다.
