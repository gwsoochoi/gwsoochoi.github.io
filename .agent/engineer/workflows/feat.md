# Feat

> 새 섹션, 새 콘텐츠 구조, 새 UI 블록을 추가할 때 쓴다.

---

## 절차

1. `check` 수준으로 관련 파일을 읽는다.
2. 필요하면 brief 또는 issue를 기준으로 완료 조건을 고정한다.
3. 변경 파일과 검증 방법을 먼저 정리한다.
4. 최소 변경으로 구현한다.
5. `npm run lint`, `npm run build`를 실행한다.
6. 3개 locale과 모바일 폭을 수동 확인한다.
7. 필요하면 `ship`으로 마감한다.

---

## 주의점

- About/Resume는 정보량이 많으므로 시각적 위계가 무너지지 않는지 같이 본다.
- 새 텍스트 블록은 번역 위치(JSON vs TS 데이터)를 먼저 결정한다.
- 라우팅이나 새 페이지 추가 시 static export 경로 생성을 확인한다.
