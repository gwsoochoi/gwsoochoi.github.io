# Refactor

> 동작은 유지한 채 구조, 가독성, 재사용성을 개선할 때 쓴다.

---

## 절차

1. 현재 동작을 먼저 고정한다.
2. 어떤 중복/복잡도를 줄일지 정한다.
3. 새로운 추상화가 정말 필요한지 확인한다.
4. 작은 단계로 분리해 수정한다.
5. lint/build와 주요 페이지 수동 확인으로 동작 유지 여부를 본다.

---

## 이 저장소에서 특히 유효한 리팩터링

- 긴 JSX 블록의 렌더링 조각 분리
- `career.ts` 데이터 shape 정리
- locale별 중복 문자열/조건문 정리
- Tailwind 클래스 중복 축소
