# Briefs

> 기능 사양서까지는 아니지만, 여러 파일에 영향을 주는 작업을 정리하는 경량 문서 공간.

---

## 언제 만든다

- 하나의 변경이 `page + content + messages`처럼 여러 소스를 동시에 건드릴 때
- UI 개편 목표와 완료 조건을 먼저 고정해두고 싶을 때
- 나중에 같은 요청을 반복 설명하지 않으려 할 때

---

## 권장 형식

- 파일명: `YYYYMMDD-scope-title.md`
- 템플릿: [`_template.md`](_template.md)

예시:

```text
20260422-resume-density-tuning.md
20260422-about-hero-refresh.md
```
